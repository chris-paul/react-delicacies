### bable配置说明

```javascript
/* 是webpack的插件,可以调用babel-core中的API对代码进行转译 */
npm install babel-loader --save-dev
/* 把 js 代码分析成 ast (抽象语法树, 是源代码的抽象语法结构的树状表现形式)，方便各个插件分析语法进行相应的处理。有些新语法在低版本 js 中是不存在的，如箭头函数，rest 参数，函数默认值等，这种语言层面的不兼容只能通过将代码转为 ast，再通过语法转换器分析其语法后转为低版本 js */
npm install @babel/core --save-dev
/* 这个时候npm run build还不能识别react的jsx的语法，
打包的时候会失败(ReactDOM.render(<App/>, document.body)根本就没办法识别,必须在babelrc中进行配置,让babel可以以JSX和ES5的标准进行编译 */
npm install @babel/preset-env --save-dev
npm install @babel/preset-react --save-dev
/* 首先，这个配置文件是针对babel 6的。Babel 6做了一系列模块化，不像Babel 5一样把所有的内容都加载。比如需要编译ES6，我们需要设置presets为"es2015"，也就是预先加载es6编译的相关模块，如果需要编译jsx，需要预先加载"react"这个模块 */

/*当babel帮我们编译了es6语法之后，常常还会遇到了这样的错误提示，比如我们在项目中运用了async/await。这时我们就需要@babel/polyfill为我们在全局去注入这些ES6+的变量（或者属性/方法）*/
npm install @babel/polyfill --save-dev
/* 所有帮助程序都将引用该模块,运行时将编译到我们的构建中,去掉bael的一些全局变量 */
npm install @babel/plugin-transform-runtime --save-dev
/* 解析类的属性例如static等属性 */
npm install @babel/plugin-proposal-class-properties --save-dev
/* ES7 Decorator */
npm install @babel/plugin-proposal-decorators --save-dev
/* 支持异步import */
npm install --save-dev @babel/plugin-syntax-dynamic-import
/* anted 按需加载 */
npm install babel-plugin-import --save-dev
/* ESLint 核心规则不支持实验阶段的语法，如果要使用这部分语法，需要配合 eslint-plugin-babel 使用 */
npm install eslint-plugin-babel --save-dev
/* 规范markdown */
npm install --save-dev eslint eslint-plugin-markdown
```

### react redux 依赖
```javascript
/* 安装react */
npm install react --save
/* 安装react-dom,将react虚拟DOM渲染成真实的DOM */
npm install react-dom --save
/* store管理器redux,react-redux集成了包括connect等等很多方法 */
npm install redux --save
/* 提升数组对象操作性能 */
npm install immutable --save
/* 更高效的创建reducer */
npm install redux-immutablejs --save
/* 简化了我们编写组件的方式,例如connect创建傻瓜组件等 */
npm install react-redux --save
/*react-router包含了react-router-dom和react-router-native*/
npm install react-router --save
npm install react-router-dom --save
/*跟路由相关的跳转包*/
npm install history --save
/* react重写每一个组件的HMR代码插件 */
npm install react-hot-loader --save-dev
/* 需要处理异步请求,相比于thunk需要自己dispatch,promise又瘦身过度,所以两个中间件处理redux异步调用 */
npm install redux-thunk --save
```
